Intrinsically Typed ASTs

Posted on November 21, 2022

We process programming language text by first parsing it into an internal data structure called an abstract syntax tree (AST), we then interpret the AST (in a broad sense) by traversing it (possibly in several stages) to produce some kind of output, be it executable machine code or byte code (we call this compilation), or a value in the language (we call this interpretation in the narrow sense).

Not all input programs that are syntactically allowed, necessarily make sense and should be accepted. For example, we may want to reject programs that reference undefined variables, or we may want to enforce a type discipline.

Better solution: the result of type checking the AST should be an AST with type information, i.e., an intrinsically typed AST. The interpreter becomes more elegant.
